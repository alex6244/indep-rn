```
Ты — senior-инженер (React Native / Expo, TypeScript, Node API). Проведи **полный аудит** репозитория `indep-rn` с упором на **архитектуру, качество кода, поддерживаемость и безопасность**. Не предлагай переписать проект с нуля; давай конкретные находки с путями к файлам.

### Контекст проекта
- **Мобилка:** Expo Router, `src/app`, `src/features`, `src/services`, `src/shared`, `src/widgets`
- **ИИ-подбор:** `src/features/aiPicker/` (правила, каталог, UI), маршрут `/ai-picker`
- **Бэкенд ИИ (отдельная папка):** `ai-api/` (Hono, `POST /v1/chat`, `POST /v1/leads`, каталог с indep.su)
- **Вне скоупа:** корпоративный Laravel (`nezavis/indep`, `indep.git`) — не анализировать и не предлагать правки там
- **Документы:** `docs/AI-ROADMAP.md`, `docs/BACKEND-CARS.md`, `ai-api/README.md`
- Каталог авто: `get-cars-to-banners`, ~248 позиций; LLM (DeepSeek) планируется только в `ai-api`

### Как работать
1. Сначала **обзор структуры** (дерево `src/`, `ai-api/`, `package.json`, конфиги Expo/TS/Jest).
2. Затем **углублённый разбор** по областям ниже — читай реальный код, не делай выводов по одному файлу.
3. Запусти, где уместно: `npm run typecheck`, `npm run lint`, `npm test` (корень и `ai-api` при наличии скриптов). Зафиксируй падения.
4. Ищи **дублирование** между `src/features/aiPicker` и `ai-api/src` (логика чата/фильтров).
5. Не трать время на `node_modules`, `.expo`, сгенерированные ассеты, `design/figma/raw/`.

### Области проверки (все обязательны)

#### A. Архитектура и границы слоёв (ПРИОРИТЕТ)
- Соблюдается ли разделение: `app` (маршруты) → `features` (сценарии) → `services` (API/данные) → `shared` (UI/утилиты)?
- Нет ли «божественных» экранов, бизнес-логики в UI, прямых `fetch` из компонентов в обход сервисов?
- Консистентность feature-slices: `ui/`, `hooks/`, типы, тесты рядом с фичей?
- Expo Router: корректность `_layout`, deep links, лишние re-export?
- **ai-api:** отделение routes / catalog / chat; что вынести в shared-пакет, чтобы не расходились две копии правил?
- Зависимости между фичами (циклы, features → features без абстракции)?
- Стратегия env (`src/config/env.ts`, `EXPO_PUBLIC_*`): утечки секретов, динамический доступ к env?

#### B. Чистота кода и TypeScript (ПРИОРИТЕТ)
- Строгость типов: `any`, небезопасные cast, дубли типов (app vs ai-api).
- Именование, размер функций/файлов, повторяющиеся паттерны.
- Обработка ошибок: пустые `catch`, проглатывание ошибок сети, сообщения пользователю (`getUserErrorMessage` и аналоги).
- Async: race conditions в `useEffect`, отмена запросов, состояние loading/error.
- Соответствие существующему стилю (импорты, структура, уровень комментариев).
- Мёртвый код, закомментированные блоки, неиспользуемые экспорты.

#### C. React Native / Expo
- Производительность списков (`FlatList`, `keyExtractor`, мемоизация).
- Безопасные зоны, клавиатура, доступность (`accessibilityLabel`).
- Изображения с CDN (кеш, плейсхолдеры, ошибки загрузки в `AiCarSuggestionCard` и каталоге).
- Platform-specific код без лишнего дублирования.

#### D. Данные и API
- `services/*`: единый HTTP-клиент, таймауты, retry, mock vs api (`EXPO_PUBLIC_*_SOURCE`).
- Схемы/валидация (zod или аналоги) на границе API.
- **aiPicker:** загрузка каталога (api vs seed), fallback, рассинхрон с `ai-api`.
- Контракт с будущим `GET /cars` (`docs/BACKEND-CARS.md`) — зазоры в типах/id.

#### E. ИИ-подбор (продукт + техника)
- `ruleBasedReply`, `parseUserIntent`, `filterCatalog`: краевые случаи (непонятный запрос, fallback на «первые 5 LADA»).
- Дублирование логики app ↔ ai-api.
- Готовность к DeepSeek: куда вставить, что оставить rule-based, риск галлюцинаций.
- Лиды: валидация телефона, PII, отсутствие продакшен-канала.
- CORS, rate limit, отсутствие auth в чате — осознанные риски?

#### F. Тесты
- Покрытие критичных путей (auth, catalog, aiPicker, services).
- Качество тестов (не тавтология).
- Пробелы: что добавить в первую очередь (3–5 тест-кейсов списком).

#### G. Безопасность
- Секреты в репо, `.env`, логи с телефонами в `__DEV__`.
- `EXPO_PUBLIC_*` — что нельзя светить.
- ai-api: CORS `*`, `GET /v1/leads` в dev, инъекции в промпт (когда появится LLM).

#### H. Сборка, CI, DX
- `package.json` scripts, typecheck, lint, Detox/e2e если есть.
- `.gitignore`, лишние untracked, размер репо.
- Документация vs реальность (роадмап, README ai-api).

#### I. Продукт / UX (кратко)
- Понятность ИИ-чата без LLM.
- Индикатор источника каталога (api/seed).
- Согласованность с `docs/AI-ROADMAP.md`.
```
