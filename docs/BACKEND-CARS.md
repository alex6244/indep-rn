# Каталог автомобилей — контракт для бэкенда

Базовый URL: значение `EXPO_PUBLIC_API_URL` (сейчас `https://indep.su/api/v1.0`).

Авторизация: для каталога желателен `Authorization: Bearer <access_token>` (при 401 приложение покажет «Войдите в аккаунт»).

Формат ответа: **массив / объект напрямую** или обёртка `{ "data": ... }` — мобильное приложение поддерживает оба варианта.

Content-Type: `application/json`.

---

## `GET /cars` — список

**Назначение:** каталог, избранное (подгрузка карточек по id), фильтры (пока часть фильтров на клиенте; query можно расширять).

### Query-параметры (уже заложены во фронте)

| Параметр | Тип | Пример | Комментарий |
|----------|-----|--------|-------------|
| `brand` | string | `BMW` | подстрока по марке (сейчас mock; уточнить семантику) |
| `bodyType` | string | `Кроссовер` | точное совпадение |
| `minPrice` | number | `1000000` | |
| `maxPrice` | number | `5000000` | |
| `page` | number | `1` | пагинация (опционально) |
| `limit` | number | `20` | размер страницы (опционально) |

Дополнительные фильтры (пока только на клиенте; для v2 API): `yearFrom`, `yearTo`, `mileageFrom`, `mileageTo`, `engineType`, `transmission`, `driveType`, `powerFrom`, `powerTo`, `paymentType`, `features[]`.

### Ответ `200 OK`

Тело: **массив объектов** `Car` (см. ниже) или `{ "data": [ ... ] }`.

Пустой каталог: `[]`.

---

## `GET /cars/{id}` — одна машина

**Назначение:** экран `/auto/[id]`.

### Ответ `200 OK`

Один объект `Car` или `{ "data": { ... } }`.

### Ошибки

| HTTP | Когда |
|------|--------|
| `404` | машина не найдена |
| `401` | нет / невалидный токен (если эндпоинт защищён) |
| `5xx` | ошибка сервера |

Тело ошибки: желательно JSON с полем `message` (строка, без HTML), до ~220 символов.

---

## Объект `Car` — поля

### Обязательные

| Поле | Тип | Пример | UI |
|------|-----|--------|-----|
| `id` | string | `"car_42"` | id, избранное, deep link |
| `title` | string | `"BMW X5 xDrive40i"` | заголовок |
| `brand` | string | `"BMW"` | фильтр, поиск |
| `price` | number | `4200000` | цена в **рублях**, целое |
| `mileage` | number | `120000` | пробег, **км** |
| `year` | number | `2022` | год выпуска |
| `engine` | string | `"3.0"` | объём двигателя (л), в строке характеристик |
| `power` | number | `249` | мощность, **л.с.** |
| `driveType` | string | `"4WD"` | код привода (`4WD`, `2WD`, …) |
| `address` | string | `"г. Москва, ул. Пример"` | адрес под карточкой |
| `images` | string[] | `["https://…/1.jpg"]` | HTTPS URL фото, минимум 0 элементов |

### Рекомендуемые (для строки «пробег — двигатель — КПП — топливо — привод»)

| Поле | Тип | Пример | UI |
|------|-----|--------|-----|
| `driveLabel` | string | `"Полный"` | подпись привода; если нет — фронт мапит `4WD`→«Полный», `2WD`→«Передний» |
| `transmission` | string | `"AT"` | коробка (AT, MT, CVT, …) |
| `fuelType` | string | `"Бензин"` | тип топлива |

### Опциональные (фильтры и бейджи)

| Поле | Тип | Допустимые значения |
|------|-----|---------------------|
| `bodyType` | string | `"Седан"`, `"Кроссовер"`, `"Хэтчбек"` |
| `features` | string[] | произвольные подписи, напр. `"Без ДТП"`, `"На гарантии"` |
| `paymentType` | string | `"cash"` \| `"credit"` |
| `hasDiscount` | boolean | скидка |
| `vatReturn` | boolean | возврат НДС |
| `weeklyOffer` | boolean | предложение недели |

Дополнительные поля в JSON **не ломают** приложение (игнорируются).

## Как это отображается в приложении

**Карточка каталога:**

- цена;
- строка: `{mileage} км - {engine} {transmission} {power} л.с. - {fuelType} - {driveLabel}`;
- строка: `{title}, {year}`;
- галерея `images`;
- адрес `address`.

**Экран авто:** заголовок, цена, адрес, галерея, год / пробег / двигатель / КПП / привод.

## Чеклист готовности для фронта

- [ ] `GET /cars` отдаёт массив с **всеми обязательными** полями
- [ ] `GET /cars/{id}` отдаёт один объект или 404
- [ ] `price`, `mileage`, `year`, `power` — числа, не строки
- [ ] `images` — массив строк URL (желательно HTTPS)
- [ ] Стабильный `id` для избранного (`POST/DELETE /favorites/{carId}`)

После проверки на staging: в приложении `EXPO_PUBLIC_CATALOG_SOURCE=api` и прогон сценария из `docs/DEMO.md` (каталог → карточка → избранное).
